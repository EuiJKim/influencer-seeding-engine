# 온톨로지 + 점수화 리허설 (상태 0→1) — 2026-07-16

목적: 대표가 말한 1번 과제 "온톨로지 구축"과 시딩 엔진의 첫 판단(누구에게 보낼까)을
실데이터로 한 바퀴 검증. 당일은 이 구조에 현장 요구사항·데이터를 꽂는다.

## 1. 온톨로지 스키마 (v0 — 리허설판)

"모든 데이터를 못 가져오니 무엇을 선택하느냐가 문제 정의" → 각 필드에 **획득 가능성 라벨**을 붙였다.
라벨: ●확정(공개데이터로 지금 수집됨) ◐추정(간접 신호) ○당일보강(현장 데이터·설계 필요)

```
Influencer (인플루언서)
  ● channel, channel_url        — 수집됨 (yt-dlp)
  ● subscribers                 — 수집됨 (16건 중 15건, 1건 미확인→추정 플래그)
  ◐ category(뷰티 적합도)        — 제목 키워드로 추정 (R4)
  ◐ authenticity(내돈내산 성향)   — 제목 마커로 추정 (R3)
  ○ 과거 시딩 회신율              — 브랜드 내부 데이터
  ○ 인스타 계정 연동              — 로그인 장벽, 당일 수동/제공 데이터

Content (콘텐츠)
  ● title, views, likes, comments, upload_date, duration
  ◐ 협찬 여부                    — 제목 표기로 추정 (표시광고법 표기 활용)
  ○ 댓글 감성                    — 당일 여유 시 추가

Product (제품)
  ● 라인 파악됨: 마스크/세럼/크림/디바이스 (03 문서)
  ○ 마진·재고·시딩 원가           — 브랜드 내부 데이터

SeedingAction (시딩 행위) — 당일 설계의 핵심
  ○ 전용코드, 전용링크, 발송일, 게시 확인

ConversionSignal (전환 신호) — 인과 측정 지점
  ○ 코드 사용 구매(확실) / 게시 후 시간창 매출 상승(추정) — 두 칸 분리 유지
```

## 2. 점수 규칙 (R1~R4, 합계 100)

정의: [data/seeding-rulepack.json](../data/seeding-rulepack.json) · 실행: [tools/score_influencers.py](../tools/score_influencers.py)
예선 린터와 같은 원칙 — 규칙마다 근거·출처, 순수 함수, 확정/추정 분리.

| 규칙 | 배점 | 내용 | 근거 |
|---|---|---|---|
| R1 반응률 | 40 | (좋아요+댓글)/조회수, 구간제 + **최소 표본 500뷰 가드** | 리파 캠페인 사례 |
| R2 규모 적합 | 25 | 1만~10만(마이크로) 최고점, 메가 감점 | 픽플리·성공사례 |
| R3 진정성 | 20 | 내돈내산 마커 > 중립 > 협찬 표기 | 체험단 실무 분석 |
| R4 카테고리 | 15 | 뷰티 키워드 일치 여부 | 인지 채널 조사 |

셀프테스트 7/7 PASS (결정론 검증 포함).

## 3. 실데이터 실행 결과 (메디테라피 언급 15개 채널)

상위: 글램미 72점(1.4만 구독, 반응률 2.8%) > 부지런한딩 63점(207 구독, 반응률 3.0%, 내돈내산) > …
전체: [data/influencer-scores.csv](../data/influencer-scores.csv)

시스템이 "발견"한 것 (심사 스토리 재료):
1. **유료 커머스쇼(선데이미디어, 2.8만 구독)의 반응률 0.2% → 8위.** 구독자 규모로 뽑았다면 1순위였을 채널이 반응 기준으로는 하위권 — "감이 아니라 규칙이 판단"의 실증
2. 200명대 나노 채널(부지런한딩)이 3% 반응률로 2위 — 마이크로/나노의 효율 가설과 일치
3. 구독자 미확인 채널은 자동으로 "(추정)" 표기 — 확정/추정 분리 원칙 작동

## 4. 리허설에서 잡은 결함 (당일 반복 금지 목록)

1. **소표본 왜곡**: 조회수 2회·반응률 50%로 1위를 차지한 사례 발생 → `min_views: 500` 가드 추가 + 회귀 셀프테스트. 당일 다른 지표(팔로워 급증 등)에도 같은 함정 주의
2. 채널 중복: 한 채널이 여러 검색어에 걸림 → 채널 단위 대표 영상(최고점)으로 dedupe
