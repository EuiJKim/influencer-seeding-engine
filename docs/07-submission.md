## 산출물 1. 시스템 설계서

### 전체 그림 (파이프라인)

```
[수집] 카테고리 검색으로 후보 풀 확보 (유튜브 공개 지표 — 실수집 20채널 완료)
   ↓
[자격] 구독 5,000 미만·미확인 채널 제외 (비교 성립 조건)
   ↓
[판단] R1~R4 점수화: 반응률(40)·규모적합(25)·진정성(20)·카테고리(15)
   ↓
[배치] 활용 4명(효율×총량 상위) + 탐색 1명(정보 가치 예산 20%)
   ↓
[실험] 인플루언서별 전용코드·링크 발급 + 제품 발송 → 게시
   ↓
[증명] 인과 장부: 확실(코드 리딤 매출) | 추정(게시 후 48h 매출 상승) — 절대 안 섞음
   ↓
[재구매] 코드 첫 구매 고객을 코호트로 30/60일 재구매 추적 (시딩의 진짜 ROI = LTV)
   ↓
[갱신] new = 0.5×사전점수 + 0.5×실측 백분위 → 재계약/유지/제외
   ↓
[확장] 검증된 에이스와 R1~R4 프로필이 닮은 미시딩 채널 발굴 → 다음 배치
   (루프: 배치가 돌 때마다 판단 근거가 실측으로 교체된다)
```

### 각 판단의 근거 (판단 지점 × 근거 표)

| 판단 지점 | 규칙 | 근거 |
|---|---|---|
| 왜 유튜브·인스타 중심 | 퍼널 역할 분담 | 스킨케어 인지 채널 유튜브 31.4%+인스타 28.7% (픽플리 26-1Q) |
| 자격 하한 5,000 | pool_floor | 나노 채널은 반응률 표본 불안정 + 절대 도달 무의미 (설정값, 조정 가능) |
| R1 반응률 40점 | (좋아요+댓글)/조회, 500뷰 미만 무효 | 마이크로 반응률→전환 국내 사례 (리파: 3개월 일매출 2배) |
| R2 규모 25점 | 1만~10만 최고점 | 시딩(무상)은 메가 회신율 낮음 — 실수집에서도 재현 (268만 채널 반응률 1.2%) |
| R3 진정성 20점 | 내돈내산>중립>협찬 표기 | 커머스쇼 0.2% vs 내돈내산 3.0% — 15배 차이 (실수집) |
| R4 카테고리 15점 | 뷰티 키워드 | 카테고리 밖 도달은 전환 낮음 |
| 가중치 숫자 자체 | 초기값 | 민감도 검증: 5개 대안 가중치 전부에서 top5 유지 3채널 — 결론은 가중치에 강건. 루프가 실측으로 보정 |
| 배치 4+1 | 활용+탐색 | 단일 라운드는 근접 채널 구분 불가(시뮬레이션으로 확인) → 해상도 올릴 표본 필요 |
| 확실/추정 분리 | 코드=확실, 시간창=추정 | 전역 어트리뷰션 불가(반사실·파편화·유인충돌) → 1st-party 실험 설계로 전환 |

### 온톨로지 (평가 1 대응) — 소비자 상태 세계

세계의 중심 = 소비자의 마음 상태. 자극(콘텐츠·후기·코드)이 상태를 전이시킨다.

```
[자극층]  ①인플루언서 콘텐츠  ②검증 콘텐츠(후기·블로그·리뷰)  ③전용코드
[소비자]  모름 → 인지 → 관심 → 신뢰 → 구매 → 재구매 → 전파
[관측]    ◐간접(조회·좋아요)  ✗불가(구전·오프라인 — 추정하지 않고 명시)  ●확실(코드 리딤)
[시스템]  시딩 엔진 = 자극 선택기  /  인과 장부 = 상태 변화 기록계 (append-only)
세계의 자체 루프: 전파(후기·입소문)가 다음 소비자의 검증 자극이 된다
```

관측 불가 구간을 채워넣지 않고 "불가"로 명시한 것이 이 온톨로지의 정직성이다.
브랜드 비종속 설계 — 카테고리 키워드만 교체하면 헤어·향수(멀티브랜드)로 이식된다.

### 인과관계 (평가 2 대응)

- 전역 멀티터치 어트리뷰션은 3중 구조로 불가능(반사실 관측 불가·관측 주체 파편화·측정 유인 충돌)
- 접근 전환: 관측이 아니라 **실험 설계** — 인플루언서별 전용코드 = 채널 단위 실험
- 확실(코드 리딤)과 추정(시간창 상승)을 절대 섞지 않는 이중 장부
- 한계 명시: 시간창 추정치는 게시 겹침 시 과대계상 가능 / 코드 미사용 구매는 하한 추정만 가능

### 재현성 (평가 3 대응)

- 전 판정이 순수 함수(같은 입력=같은 출력), 규칙은 rulepack.json에 근거·출처와 함께 고정
- 셀프테스트 22종 PASS: 점수화 10(소표본 왜곡 가드·민감도·풀 자격 포함) + 루프 7(정답 심고 복원 검증) + 닮은꼴 5
- 원커맨드 재현: `python tools/run_demo.py` — 검증→판단(실데이터)→장부→루프→대시보드 자동 생성
- 매출 데이터는 [SYNTHETIC] 라벨 명시 — 합성에 정답을 심고 시스템이 복원하는지로 추정 로직 자체를 검증

## 산출물 2. 매출가설

### 가설문

> **H1.** 구독 5천~40만의 뷰티 전문 채널 가운데 반응률 1.5% 이상·내돈내산 성향 채널에
> 전용코드와 함께 시딩하면, 코드 리딤으로 측정 가능한 직접 매출이 발생하며,
> 그 크기는 채널의 반응 절대량(좋아요+댓글)에 비례한다.
>
> **H2.** 코드로 유입된 첫 구매 고객의 30일 재구매율은 일반 유입 고객보다 높다
> (신뢰 자극을 거쳐 온 고객이라 LTV가 높다) — 시딩의 진짜 ROI는 첫 구매가 아니라 재구매에서 난다.
>
> **H3.** 협찬 표기형 대형 채널은 도달 대비 리딤이 낮다 (진정성 결핍은 전환 단계에서 페널티).

### 무엇을 근거로 한 가설인가

1. **실수집 데이터 (2026-07-17, 유튜브 공개 지표 20채널)** — 반응률 분포 0.1%~3.1%로 30배 격차:
   채널 선택이 결과를 가르는 폭이 실증됨. 협찬 커머스쇼 0.2% vs 내돈내산 후기 3.0% (15배).
   전문가형 채널(피부과 의사 운영)이 반응률 2.1~2.2%로 상위 — 신뢰가 반응을 만든다.
2. **업계 사례** — 마이크로 인플루언서 중심 캠페인으로 3개월 만에 일매출 2배 (리파, 국내).
3. **제도적 증거** — 올리브영이 코드형 어필리에이트(쇼핑 큐레이터)를 상시 운영:
   코드 리딤 기반 성과 측정이 실무에서 작동한다는 증거.
4. **소비 여정 조사** — 스킨케어 인지의 60%가 유튜브+인스타에서 발생 (픽플리 26-1Q).

### 시나리오 (4주, 1차 배치) — 수치는 [SYNTHETIC] 시뮬레이션

배치: 실수집 랭킹 기반 5채널 — 닥터블랑(18.8만, 의사), Jaina(28.2만), 소살(13.5만),
일타의사 박영진(22.6만) [이상 활용 4] + 림온(15.9만) [탐색 1]

```
D+0   전용코드 5종 발급(MEDI-○○○, 10% 할인) + 제품 발송 + DM 컨택
D+3~7 게시 확인 (게시일이 시간창 측정의 기준점)
D+7~  자사몰 관리자에서 코드 리딤 주 단위 기록
W4    인과 장부 마감 — 시뮬레이션 결과(합성):
        코드 리딤 12~47건/채널, 직접 매출 36만~141만 원
        사전점수 1위 채널이 실측 하위로 강등, 중위 채널이 에이스로 판명
        → 갱신 점수로 2차 배치 + 에이스 닮은꼴 2채널 신규 투입
W4+   코드 첫 구매 코호트의 30일 재구매 추적 개시 (H2 검증)
```

보수/기대 시나리오 병기: 리딤이 0에 수렴해도 실험은 실패가 아니다 —
"이 유형은 매출이 안 난다"는 확정 정보가 남고, 다음 배치의 근거가 된다 (탐색의 가치).

### 가설이 틀렸는지 어떻게 아는가 (반증 조건)

- H1 반증: 반응 절대량 상위 채널의 리딤이 하위 채널보다 유의하게 낮으면 기각 → R1 가중치 하향 보정
- H3 반증: 협찬형 대형 채널 리딤이 내돈내산형보다 높으면 기각 → R3 재설계
- 반증되면 룰팩 가중치가 갱신된다 — 가설이 틀리는 것까지 루프의 연료다.

## 산출물 3. 이관 가능성 — 메디테라피 팀이 내일 하는 일

개발자 없이(현재 조직 상태 반영), 마케터 1명 · 주 2~3시간으로 즉시 실행 가능하다.

### Day 1 (내일)
1. 자사몰 관리자에서 쿠폰 코드 5종 생성 — `MEDI-{채널명}` 형식, 10% 할인, 유효 4주
2. 시딩 대장 스프레드시트 생성 — 열: 채널·코드·발송일·게시일·주차별 리딤·비고
   (본 제출물의 CSV 스키마 그대로: influencer-scores.csv / attribution-ledger.csv)
3. 1차 배치 5채널 확정 — 첨부 랭킹의 실명 채널 리스트 그대로 사용 가능 (전부 공개 채널)

### Day 2~3
4. DM 컨택 — 템플릿: "제품 무상 제공 + 전용 10% 코드(팔로워 혜택) 제공, 게시 의무 없음,
   협찬 표기는 표시광고법에 따라 진행" (진정성 훼손 최소화 조건)
5. 제품 발송 (발송일 대장 기록)

### Week 2~3
6. 게시 확인 → 게시일 기록 / 자사몰 관리자 매출 화면에서 코드 리딤 주 단위 기록
7. 게시 후 48시간 자사몰 매출을 직전 4주 같은 요일 평균과 비교 → "추정" 칸에 별도 기록
   (확실 칸과 절대 합산 금지 — 본 설계의 핵심 규율)

### Week 4
8. 장부 마감 → 갱신식 적용: 새 점수 = 0.5×사전점수 + 0.5×(리딤 실적 백분위)
   70점 이상 재계약+닮은꼴 발굴 / 45~69 유지 / 45 미만 제외
9. 코드 첫 구매 고객 코호트의 30일 재구매율 추적 개시
10. 2차 배치 구성 — 에이스 닮은꼴(첨부 lookalikes 방식) + 탐색 1명 유지

### 도구 이관
- 파이썬 도구 5종 동봉 (수집·보강·점수화·시뮬레이션·대시보드) — 외부 의존성 없음(stdlib),
  파이썬만 설치하면 `python tools/run_demo.py` 한 줄로 재현
- 파이썬이 없어도 됨: 규칙(rulepack.json)의 표를 스프레드시트 수식으로 옮기면 동일 판정
- 확장: 카테고리 키워드만 교체하면 헤어·향수 등 차기 브랜드에 동일 엔진 적용

### 정직한 한계 (도입 전 알아야 할 것)
- 인스타그램 지표는 자동 수집 불가(로그인 장벽) — 수동 기록 or 공식 API(비즈니스 계정) 필요
- 시간창 추정치는 다른 마케팅과 겹치면 과대계상 가능 — 확실(코드)만 의사결정 근거로 사용 권장
- 단일 라운드로는 근접 채널 우열 확정 불가 — 최소 2~3라운드 반복이 전제
