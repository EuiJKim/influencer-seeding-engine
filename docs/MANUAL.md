# 시딩 엔진 운영 매뉴얼 — 마케터 1인용

개발자 없이 혼자 세팅하고 운영하는 방법. 코드 지식 불필요.
소요: 최초 세팅 30분, 이후 월초 1시간 + 매주 금요일 15분 + 월말 1시간.

---

## 0. 최초 세팅 (1회, 30분)

1. **폴더 받기** — 이 저장소 전체를 다운로드(GitHub 초록 버튼 Code → Download ZIP)해서
   `C:\시딩엔진` 같은 곳에 압축 해제.
2. **파이썬 설치** — python.org → Downloads → 설치 시 **"Add Python to PATH" 체크박스 반드시 체크**.
3. **수집 도구 설치** — 윈도우 검색에서 PowerShell 열고:
   ```
   pip install yt-dlp
   ```
4. **동작 확인** — PowerShell에서 (따옴표 포함 그대로):
   ```
   cd "C:\시딩엔진"
   python tools\run_demo.py
   ```
   "10/10, 7/7, 5/5 passed"가 나오고 브라우저에 대시보드가 뜨면 세팅 끝.

> 명령 규칙 딱 2개: ① 명령은 항상 엔진 폴더에서 친다(cd 먼저) ② 파이썬 파일은 항상 `python ` 을 앞에 붙인다.

## 1. 월초 루틴 — 후보 수집과 배치 확정 (1시간)

### 1-1. 후보 수집 (명령 2개, 10분)

제품에 맞는 검색어로 바꿔서 실행 (예: 앰플 출시라면 "앰플 추천"):
```
python tools\collect_youtube.py --out=data/youtube-candidates.csv "선크림 추천" "선크림 리뷰" "스킨케어 루틴 추천"
python tools\enrich_youtube.py --src=data/youtube-candidates.csv --out=data/youtube-enriched-candidates.csv --all
```
검색어 선정 원칙: 브랜드명 말고 **소비자가 구매 직전에 검색하는 카테고리 단어** + "추천/리뷰".

### 1-2. 점수·배치 받기 (명령 1개)

```
python tools\run_demo.py
```
브라우저 대시보드에서 볼 곳:
- **① 판단**: 20채널 랭킹. 각 행에 점수 근거가 붙어 있고, 막대에 마우스를 올리면 상세.
- **배치 5명**: 활용 4(점수·반응량 상위) + 탐색 1(실험용). 이게 시스템의 제안.

### 1-3. 사람의 검토 (30분) — 시스템이 못 하는 부분

상위 5명 채널에 실제로 들어가서 확인:
- [ ] 최근 영상 톤이 브랜드와 맞는가
- [ ] 논란 이력이 없는가
- [ ] 경쟁사 전속 협찬 중이 아닌가
문제 있으면 그 채널을 빼고 랭킹 6위를 올린다. 랭킹 20명 = 내장 대기명단.

### 1-4. 쿠폰 생성 + 시딩 대장 만들기

- 자사몰 관리자 → 쿠폰/프로모션 → 신규 5종: 코드명 `MEDI-{채널명}`, 10% 할인, 유효 4주.
- 엑셀로 "시딩 대장" 생성 — 열:
  `채널 | 코드 | 발송일 | 게시일 | 1주리딤 | 2주 | 3주 | 4주 | 누적(확실) | 추정(48h) | 비고`
  **누적(확실)과 추정 열은 절대 합산하지 않는다** — 이 분리가 이 시스템의 핵심 규율.

### 1-5. DM 컨택 (템플릿)

> 안녕하세요, 메디테라피입니다. ○○님 콘텐츠를 보고 저희 신제품을 무상으로 보내드리고 싶어
> 연락드립니다. 게시 의무는 없고, 원하시면 팔로워분들 전용 10% 할인코드(MEDI-○○)를 함께
> 안내드릴 수 있습니다. 게시하시는 경우 협찬 표기는 표시광고법에 따라 부탁드립니다.

거절은 흔하다(무상 시딩 회신율 30~50%). 거절분은 랭킹 다음 순위로 채워 재컨택.
수락자에게 제품 발송, 대장에 발송일 기록.

## 2. 주간 루틴 — 매주 금요일 15분

1. 각 채널 게시 여부 확인 → 게시일 기록 (최초 1회)
2. 자사몰 관리자 → 쿠폰 사용 내역 → **코드별 사용 건수**를 대장의 해당 주차 칸에 기록
3. (선택) 게시 후 48시간 자사몰 일 매출을 직전 4주 같은 요일 평균과 비교해 "추정" 칸에 메모

## 3. 월말 루틴 — 결산과 다음 배치 (1시간)

### 3-1. 점수 갱신 (엑셀 수식 한 줄)

대장에 열 2개 추가: `사전점수`(대시보드 랭킹의 점수), `갱신점수`:
```
갱신점수 = 0.5 * 사전점수 + 0.5 * (누적리딤 / 배치최대리딤 * 100)
```
엑셀 수식 예 (2행 기준, I열=누적):  `=0.5*L2 + 0.5*(I2/MAX($I$2:$I$6)*100)`

### 3-2. 판정 (규칙 고정)

- 70점 이상 → **재계약** + 닮은꼴 발굴 대상
- 45~69 → 유지 (다음 배치 후보군에 잔류)
- 45 미만 → 제외
- [탐색] 채널은 성과가 낮아도 실패가 아님 — "이 유형은 안 된다"는 확정 정보를 얻은 것.

### 3-3. 다음 배치 구성

- 재계약 채널 + 대시보드 "④ 확장"의 닮은꼴 추천 채널 + 탐색 1명(항상 유지)
- 1-3의 사람 검토부터 다시 → 루프 반복. 돌 때마다 명단이 실측으로 똑똑해진다.

### 3-4. 재구매 추적 (H2 가설 검증)

코드로 첫 구매한 고객 목록(자사몰 주문 내역에서 코드 필터) → 30일 뒤 재구매 여부 확인
→ "코드 유입 고객의 30일 재구매율"을 분기별로 기록. 시딩의 진짜 ROI는 여기서 판단.

## 4. 규칙 바꾸기 — 개발자 없이

모든 판단 기준은 `data/seeding-rulepack.json` 한 파일에 한글 설명과 함께 있다.
메모장으로 열어 **숫자만** 바꾸면 됨. 예:
- 구독 하한을 2만으로: `"min_subscribers": 5000` → `20000`
- 반응률 배점 낮추기: `"weight": 40` → `30`
바꾼 뒤 `python tools\run_demo.py` 다시 실행하면 새 규칙으로 재채점.
(검증 테스트가 함께 돌므로 규칙을 잘못 건드리면 FAIL로 바로 알 수 있다)

## 6. 플랜 B — 파이썬 없이 엑셀만으로

엔진의 본체는 코드가 아니라 규칙표다. rulepack.json의 표를 엑셀 수식으로 옮기면 동일 판정:
- R1 반응률: `=IF(조회<500,0,IF(반응률>=5%,40,IF(반응률>=3%,30,IF(반응률>=1.5%,20,IF(반응률>=0.5%,10,0)))))`
- R2 규모: `=IF(구독<5000,"제외",IF(AND(구독>=10000,구독<100000),25,IF(구독>=500000,10,IF(구독>=100000,15,18))))`
- R3 진정성: 제목에 "내돈내산/솔직" 있으면 20, "광고/협찬" 있으면 8, 없으면 12
- R4 카테고리: 제목에 뷰티 단어 있으면 15, 없으면 5
채널 지표(구독·조회·좋아요)는 유튜브 채널 페이지에서 손으로 옮겨 적는다 (채널당 2분).
