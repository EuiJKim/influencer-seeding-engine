# 루프 시뮬레이션 리허설 (상태 2→6) — 2026-07-16

목적: 대표의 2·3번 과제(인과 추정, 반복 루프)를 끝까지 한 바퀴. 합성 데이터에
**정답(채널별 진짜 전환력)을 심고**, 인과 장부가 그 정답을 복원하는지로 추정 로직을 검증
— 예선 골든셋과 같은 방법론.

실행: `python tools/simulate_loop.py` (셀프테스트: `selftest`) · 6/6 PASS

## 흐름과 결과 (전부 [SYNTHETIC] — 매출·전환은 가상)

```
상태 2  점수 상위 5 채널 + 전용코드(MEDI-…) + 게시일 배정
상태 4  28일 합성 판매 로그 (기저 ~101건/일 + 코드 구매 + 게시 후 48h 상승)
상태 5  인과 장부 — 확실(코드 사용분)과 추정(시간창 상승분)을 절대 안 섞음
상태 6  new = 0.5×사전점수 + 0.5×실측 백분위 → 재계약/유지/제외 + 다음 배치
```

결과 요약: 사전점수 1위(글램미 72)는 실측 후 55로 강등(심어둔 낮은 전환력을 시스템이 발견),
사전 2위(부지런한딩)와 4위(보아따)가 실측 1·2위로 승격 — **루프가 사전 예측의 오류를 스스로 교정**.

## 리허설에서 배운 것 2가지 (심사 답변 재료)

1. **근접한 전환력(1.0 vs 0.9)은 단일 라운드로 구분 불가** — 노이즈가 격차보다 큼.
   47 vs 43건으로 근소 역전 발생. 처음엔 "순위 완전 일치"를 검증하려다 실패 →
   "뚜렷이 갈린 층위(상위 2 집합, 최하위)만 복원 요구"로 정직하게 수정.
   → **이게 루프를 반복하는 이유다**: 한 번의 시딩은 대략적 층위만 알려주고,
   라운드가 쌓일수록 해상도가 올라간다. (당일 설계 설명에 그대로 사용)
2. **추정치의 한계 명시**: 시간창 상승분은 다른 채널 게시와 겹치면 과대계상 가능 —
   장부에 주석으로 박제. 확실(코드)과 추정(시간창)을 섞지 않는 이유.
