## [필드 1] 한 줄 요약

소비자 마음 상태(모름→인지→신뢰→구매→재구매→전파)를 온톨로지로 정의하고, 인플루언서별 전용코드로 "매출이 났다"를 증명하며, 실측으로 판단 규칙이 스스로 보정되는 시딩 엔진 — 동작하는 구현 + 셀프테스트 22종 + 실수집 데이터 20채널 포함.

## [필드 2] 문제 정의

"매출이 났다"를 증명하는 문제는 곧 어트리뷰션 문제인데, 이는 어떤 회사도 전역에서 풀지 못했다. 이유는 3중 구조다. ① 반사실 관측 불가 — 같은 소비자가 "광고를 안 봤다면"의 세계는 재생되지 않는다. ② 관측 주체 파편화 — 틱톡→유튜브→네이버→오프라인 구매로 이어지는 여정을 처음부터 끝까지 보는 주체가 없고, 각 플랫폼이 기여를 중복 주장하며(합산 시 실매출의 2~3배), 이를 잇던 개인 추적은 규제(ATT·쿠키 폐지)로 차단되는 중이다. ③ 측정 유인 충돌 — 광고 효과를 재는 주체가 광고를 파는 주체다.

그래서 앵글을 바꿨다: 인과를 관측하려 하지 않고, 인과가 읽히도록 실험을 설계한다. 인플루언서마다 다른 전용코드 = 채널 단위의 실험이다. 이것이 메디테라피의 1st-party 환경에서는 성립한다. ① 자사몰 보유 — 코드가 찍히는 순간을 직접 관측(확인: meditherapy.co.kr) ② 제한된 SKU — 오염 없는 제품 단위 실험 가능 ③ 측정 유인 정렬 — 자기 돈으로 자기 효과를 재니 부풀릴 이유가 없음 ④ 시딩은 개체 단위 코드 부착이 자연스러운, 태생이 실험인 채널(올리브영 어필리에이트 제도가 실무 증거).

핵심 질문 "매출이 났다를 어떻게 보장하고 증명할 것인가"에 대한 답은 두 축이다. 보장은 사전 설계가 맡는다 — 반응률·진정성 필터로 전환 확률 높은 채널만 선정하고, 전용코드의 할인이 구매를 유인한다. 증명은 사후 측정이 맡는다 — 그 코드의 리딤이 매출을 확정 기록한다. 즉 같은 쿠폰 하나가 전환을 만들고(보장) 동시에 그 전환을 기록한다(증명) — 보장과 증명이 한 장치에서 만나며, 반복 구매는 코드 코호트 추적으로, 시스템 전체는 실측이 판단 규칙을 보정하는 루프로 닫힌다.

푼다: "누구에게 보내야 매출이 되는가"의 판단 + 코드 리딤 증명 + 재구매 추적 + 자가 보정 루프. 안 푼다: 전역 어트리뷰션(위 3중 구조로 불가능), 개인 추적(불법·불필요 — 집계 신호로 대체), 게시확인 자동화(단순 자동화라 AX가 아님). 선택 기준은 비즈니스 임팩트 × 실현 가능성 × 데이터 구득성이다.

## [필드 3] 핵심 기능·시연 포인트

원커맨드 시연: python tools/run_demo.py — 셀프테스트 22종 통과 화면 → 실데이터 점수화 → 인과 장부 → 루프 → 대시보드 자동 오픈(오프라인 동작).

시연 포인트 4개:
1. 판단의 근거가 화면에 있다 — 후보 풀 20채널(실수집)이 규칙 R1~R4로 점수화되고, 각 행에 근거가 붙는다. 268만 구독 채널이 반응률 1.2%로 최하위, 18.8만 의사 채널이 1위 — "구독자 수가 아니라 규칙이 판단"의 실증.
2. 쿠폰의 이중 역할 — 할인이 구매를 유인하고(보장), 같은 코드의 리딤이 그 매출을 확정 기록한다(증명). 장부는 확실(코드)과 추정(48h 시간창)을 절대 섞지 않는다. 초록만 "증명"이다.
3. 루프의 자가 교정 — 사전점수 1위가 실측 후 강등되고 중위 채널이 에이스로 승격. 판단 규칙 자체가 실측으로 보정된다.
4. 가중치 민감도 — 배점을 5가지 방식으로 치환해도 상위권 3채널 불변. 초기 가중치의 임의성이 결론을 흔들지 못함을 매 실행마다 증명.

## [필드 6] 한계·미구현 (선택)

- 인스타그램 자동 수집 불가(로그인 장벽) — 수동 샘플링 또는 공식 API(비즈니스 계정) 필요. 숨기지 않고 온톨로지에 관측 불가로 명시

- 단일 라운드는 근접 채널 우열 확정 불가(시뮬레이션으로 확인) — 2~3라운드 반복이 전제, 이것이 루프의 존재 이유

## [필드 7] 시스템 설계서

### 전체 그림

[수집] 카테고리 검색으로 후보 풀 확보(유튜브 공개 지표, 실수집 20채널)
→ [자격] 구독 5,000 미만·미확인 제외(비교 성립 조건, 설정값)
→ [판단] R1~R4 점수화: 반응률(40)·규모적합(25)·진정성(20)·카테고리(15)
→ [배치] 활용 4명(효율×총량 상위) + 탐색 1명(정보 가치 예산 20%)
→ [실험] 인플루언서별 전용코드·링크 발급 + 제품 발송 → 게시
→ [증명] 인과 장부: 확실(코드 리딤) | 추정(48h 시간창) — 절대 안 섞음
→ [재구매] 코드 첫 구매 고객 코호트 30/60일 재구매 추적(시딩의 진짜 ROI = LTV)
→ [갱신] 새 점수 = 0.5×사전점수 + 0.5×실측 백분위 → 재계약/유지/제외
→ [확장] 에이스와 R1~R4 프로필이 닮은 미시딩 채널 발굴 → 다음 배치
(루프: 배치가 돌 때마다 판단 근거가 실측으로 교체된다)

### 각 판단의 근거

| 판단 | 근거 |
|---|---|
| 유튜브·인스타 중심 | 스킨케어 인지 채널 유튜브 31.4%+인스타 28.7% (픽플리 26-1Q) |
| 자격 하한 5,000 | 나노 채널은 반응률 표본 불안정 + 절대 도달 무의미 (설정값, 조정 가능) |
| R1 반응률 40 | 반응률→전환 국내 사례(리파: 3개월 일매출 2배) + 500뷰 미만 표본 무효 가드 |
| R2 규모 25 | 시딩(무상)은 메가 회신율 낮음 — 실수집에서 재현(268만 채널 반응률 1.2%) |
| R3 진정성 20 | 협찬 커머스쇼 0.2% vs 내돈내산 3.0% — 15배 차이(실수집) |
| R4 카테고리 15 | 카테고리 밖 도달은 전환 낮음 |
| 가중치 숫자 | 초기값 — 민감도 검증(5개 대안 가중치에서 top5 안정) + 루프가 실측으로 보정 |
| 배치 4+1 | 단일 라운드는 근접 채널 구분 불가 → 해상도 올릴 탐색 표본 필요 |
| 확실/추정 분리 | 전역 어트리뷰션 불가 → 1st-party 실험 설계로 전환 |

### 온톨로지 — 소비자 상태 세계 (평가 1)

세계의 중심 = 소비자의 마음 상태. 자극이 상태를 전이시킨다.
[자극층] ①인플루언서 콘텐츠 ②검증 콘텐츠(후기·블로그·리뷰) ③전용코드
[소비자] 모름→인지→관심→신뢰→구매→재구매→전파 (7상태)
[관측] ◐간접(조회·좋아요) ✗불가(구전·오프라인 — 추정하지 않고 명시) ●확실(코드 리딤)
[시스템] 시딩 엔진 = 자극 선택기 / 인과 장부 = 상태 변화 기록계(append-only, 담당자가 바뀌어도 히스토리 보존)
세계의 자체 루프: 전파(후기·입소문)가 다음 소비자의 검증 자극이 된다.
관측 불가 구간을 채워넣지 않고 비워둔 것이 이 온톨로지의 정직성이다. 브랜드 비종속 — 카테고리 키워드만 교체하면 헤어·향수로 이식.

### 인과관계 (평가 2)

전역 멀티터치 어트리뷰션은 3중 구조(반사실·파편화·유인충돌)로 불가능 → 관측이 아니라 실험 설계. 인플루언서별 전용코드 = 채널 단위 실험. 확실/추정 이중 장부. 한계 명시: 시간창은 게시 겹침 시 과대계상 가능, 코드 미사용 구매는 하한 추정만 가능.

### 재현성 (평가 3)

전 판정이 순수 함수(같은 입력=같은 출력), 규칙은 rulepack.json에 근거·출처와 함께 고정. 셀프테스트 22종 PASS(소표본 왜곡 가드·민감도·풀 자격·정답 복원·결정론 포함). 원커맨드 재현: python tools/run_demo.py. 매출은 [SYNTHETIC] 명시 — 정답을 심고 복원을 검증하는 골든셋 방식.

## [필드 8] 매출가설

### 가설문

H1. 구독 5천~40만의 뷰티 전문 채널 가운데 반응률 1.5% 이상·내돈내산 성향 채널에 전용코드와 함께 시딩하면, 코드 리딤으로 측정 가능한 직접 매출이 발생하며, 그 크기는 채널의 반응 절대량(좋아요+댓글)에 비례한다.

H2. 코드로 유입된 첫 구매 고객의 30일 재구매율은 일반 유입보다 높다(신뢰 자극을 거쳐 온 고객이라 LTV가 높다) — 시딩의 진짜 ROI는 첫 구매가 아니라 재구매에서 난다.

H3. 협찬 표기형 대형 채널은 도달 대비 리딤이 낮다(진정성 결핍은 전환 단계 페널티).

### 근거

1. 실수집 20채널(유튜브 공개 지표, 2026-07-17): 반응률 0.1%~3.1%로 30배 격차 — 채널 선택이 결과를 가르는 폭의 실증. 협찬 커머스쇼 0.2% vs 내돈내산 3.0%(15배). 전문가형 채널(피부과 의사 운영)이 2.1~2.2%로 상위 — 신뢰가 반응을 만든다.
2. 업계 사례: 마이크로 중심 캠페인으로 3개월 만에 일매출 2배(리파, 국내).
3. 제도적 증거: 올리브영이 코드형 어필리에이트를 상시 운영 — 코드 리딤 성과 측정이 실무에서 작동한다는 증거.
4. 소비 여정: 스킨케어 인지의 60%가 유튜브+인스타(픽플리 26-1Q).

### 시나리오 (4주 1차 배치 — 수치는 [SYNTHETIC] 시뮬레이션)

배치: 닥터블랑(18.8만·의사)·Jaina(28.2만)·소살(13.5만)·일타의사 박영진(22.6만) [활용 4] + 림온(15.9만) [탐색 1] — 실수집 랭킹 기반 실명 채널.

D+0 전용코드 5종(MEDI-○○○, 10%) 발급 + 발송 + DM → D+3~7 게시 확인 → W1~4 자사몰에서 코드 리딤 주 단위 기록 → W4 장부 마감. 시뮬레이션 결과(합성): 리딤 12~47건/채널, 직접 매출 36만~141만 원. 사전 1위가 실측 하위로 강등, 중위가 에이스로 판명 → 갱신 점수로 2차 배치 + 닮은꼴 2채널 투입. W4+ 코드 코호트 30일 재구매 추적 개시(H2 검증).

리딤이 0에 수렴해도 실험은 실패가 아니다 — "이 유형은 매출이 안 난다"는 확정 정보가 남고 다음 배치의 근거가 된다(탐색의 가치).

### 반증 조건 (가설이 틀렸는지 어떻게 아는가)

- H1 반증: 반응 절대량 상위 채널의 리딤이 하위보다 유의하게 낮으면 기각 → R1 가중치 하향 보정
- H3 반증: 협찬형 대형 채널 리딤이 내돈내산형보다 높으면 기각 → R3 재설계
- 반증되면 룰팩 가중치가 갱신된다 — 가설이 틀리는 것까지 루프의 연료다.

## [필드 9] 이관 가능성

개발자 없이(현 조직 상태 반영), 마케터 1명 · 주 2~3시간으로 내일 시작 가능.

Day 1: ① 자사몰 관리자에서 쿠폰 코드 5종 생성 — MEDI-{채널명}, 10% 할인, 유효 4주 ② 시딩 대장 스프레드시트 생성(열: 채널·코드·발송일·게시일·주차별 리딤·비고 — 제출물 CSV 스키마 그대로) ③ 1차 배치 5채널 확정 — 첨부 랭킹의 실명 채널 그대로 사용 가능(전부 공개 채널).

Day 2~3: ④ DM 컨택 — "제품 무상 제공 + 팔로워 전용 10% 코드, 게시 의무 없음, 협찬 표기는 표시광고법 준수"(진정성 훼손 최소화 조건) ⑤ 제품 발송, 발송일 기록.

Week 2~3: ⑥ 게시 확인·게시일 기록, 자사몰 매출 화면에서 코드 리딤 주 단위 기록 ⑦ 게시 후 48시간 자사몰 매출을 직전 4주 같은 요일 평균과 비교해 "추정" 칸에 별도 기록 — 확실 칸과 절대 합산 금지(핵심 규율).

Week 4: ⑧ 장부 마감 → 갱신식: 새 점수 = 0.5×사전점수 + 0.5×리딤 백분위. 70↑ 재계약+닮은꼴 발굴 / 45~69 유지 / 45↓ 제외 ⑨ 코드 코호트 30일 재구매율 추적 개시 ⑩ 2차 배치 = 에이스 닮은꼴 + 탐색 1명 유지.

도구 이관: 파이썬 도구 5종 동봉(수집·보강·점수화·시뮬레이션·대시보드) — 외부 의존성 0, 파이썬만 설치하면 python tools/run_demo.py 한 줄로 재현. 파이썬이 없어도 rulepack.json의 규칙 표를 스프레드시트 수식으로 옮기면 동일 판정. 확장: 카테고리 키워드만 교체하면 헤어·향수 등 차기 브랜드에 동일 엔진 적용.

도입 전 알아야 할 것: 인스타 지표는 수동 기록 필요(로그인 장벽) / 시간창 추정은 타 마케팅과 겹치면 과대계상 — 확실(코드)만 의사결정 근거 권장 / 근접 채널 우열은 2~3라운드 반복이 전제.
