# 새 세션 시작 가이드

## 세션 여는 법

1. Claude Code(데스크톱 앱 또는 터미널)에서 **이 폴더**(`C:\Users\ASUS\Desktop\AX 해커톤 본선`)를 프로젝트로 열기
   - 터미널이라면: `cd "C:\Users\ASUS\Desktop\AX 해커톤 본선"` 후 `claude`
2. 아래 프롬프트를 붙여넣고 시작
3. 세션이 끝나면 `logs/claude-code/` 폴더에 로그가 자동 저장됐는지 확인

## 붙여넣을 킥오프 프롬프트

```
AX 해커톤 본선(7/18, 메디테라피 트랙) 사전 준비를 시작할게.
문제 방향은 "인플루언서 시딩 엔진"이고, 상세 요구사항은 당일 공개돼.
README.md 읽고 시작해줘. 오늘 할 일 3가지:

1. 메디테라피 기업 인터뷰 영상 분석 — https://www.youtube.com/watch?v=1KwyhyBo0Rs
   먼저 yt-dlp로 이 영상의 한국어 자막(자동 자막 포함)을 텍스트로 추출해서
   docs/에 저장하고, 그 스크립트에서 메디테라피가 말하는 시딩의 pain point와
   기대사항을 뽑아서 본선 문제 요구사항을 예측해줘.
   (yt-dlp가 없으면 설치부터: pip install yt-dlp)

2. 인플루언서 시딩 도메인 리서치 — K-뷰티 시딩 캠페인의 실제 프로세스
   (인플루언서 발굴 → 컨택 → 제품 발송 → 콘텐츠 게시 확인 → 성과 측정)를
   공개 자료 기반으로 정리하고, 기존 툴(국내외)과 그 한계, 그리고 메디테라피의
   채널/제품 현황을 조사해줘. 예선 때처럼 출처 URL을 남기고, 확인 안 되는 건
   확인 안 된다고 표시해.

3. 본선 전략 설계 — 3시간 개인전 + 브리프 제출 + 1대1 시연 + (챔피언 시)
   5분 피칭 포맷에 맞춰서: 12시 제출(선착순 심사)을 노리는 시간 배분,
   예선에서 통했던 강점(공개 근거 기반 문제정의, 결정론적 검증, 스스로 반증)을
   시딩 엔진에 어떻게 적용할지, 당일 바로 쓸 수 있는 뼈대(스캐폴드)를 어디까지
   미리 만들어둘지 제안해줘.

결과는 docs/ 폴더에 md 파일로 정리해줘.
```

## 주의

- 이 폴더에서는 API 키·비밀번호·토큰을 채팅에 절대 입력하지 않기 (로그 무편집 제출)
- 사전 준비 로그도 전부 제출 대상 — 리서치·시행착오 과정이 남는 게 오히려 평가에 유리
